# src/bot — Telegram-бот звітності

Інтерактивний бот на **grammy** (long-polling), окремий persistent-процес, який ділить `../core/` з інжест-джобом і читає ту саму БД. Запуск: `npm run bot`.

## Токен і доступ

- **Єдиний бот на все** — `@OBVCarServiceWork_bot` (новий, чистий, без webhook), токен у `TELEGRAM_BOT_TOKEN`. Той самий бот обслуговує і цей інтерактив, і вихідні алерти інжесту (`../core/telegram.js`) — `sendMessage` не конфліктує з `getUpdates`. `@obvcarservicebot` більше **не** використовується (його webhook у CarBook).
- **Owner-only**: `BOT_ALLOWED_CHAT_IDS` — comma-separated Telegram user ID (fallback на `TELEGRAM_CHAT_ID`). Порожньо скрізь → бот нікого не пускає й показує твій ID, щоб ти його додав.
- ⚠️ Бот може писати юзеру, лише якщо той **сам першим натиснув Start** у цьому боті. Тож власник має один раз відкрити `@OBVCarServiceWork_bot` і написати йому — інакше авто-звіти/алерти не дійдуть.

## Навігація (щоб меню не губилось)

- **Постійна reply-клавіатура** над полем вводу: `📊 Статистика / 🗂 Архів / 🔄 Звіт зараз / ☰ Меню` (`keyboards.js: quickKeyboard`). Кнопки шлють текст-мітку, `index.js` message:text її роутить (працює навіть посеред вводу нотатки).
- **Нативні команди** біля вводу: `/menu /stats /archive /report` (`setMyCommands` + `setChatMenuButton`).
- **Inline «« Меню»** на кожному екрані (включно з екраном аудіо).

## Меню (фічі)

1. **🔄 Звіт зараз / авто-звіти** — планувальник (`report.js`) о `BOT_REPORT_TIMES` (дефолт 13:00, 19:30, Kyiv). Період — з моменту попереднього авто-звіту (стан у `app_state.last_report_slot`/`last_report_until`, як колись у cron). На кожного менеджера — OpenAI-резюме (`analyze.js`), одним текстовим повідомленням (без PDF). "Звіт зараз" — ручний, за сьогодні, не чіпає стан розкладу.
2. **📊 Статистика менеджера** (`stats.js`) — обрати менеджера → період (день / тиждень з пн / місяць / квартал) → к-сть дзвінків, успішність %, середній бал, найчастіший слабкий етап. Кнопки: додати нотатку (`manager_notes`), переглянути нотатки.
3. **🗂 Архів розмов** (`archive.js`) — менеджер → період → список дзвінків (пагінація) → конкретний дзвінок: текст розшифровки + кнопка «🎧 Прослухати запис» (тягне свіжий record-URL з Binotel і шле аудіо в чат).
4. **❓ Поставити питання** — база знань по електронному посібнику. **Каркас** (пункт меню є, відповідає заглушкою) — підключимо, коли клієнт дасть файл посібника.

## Файли

```
index.js      — bootstrap: auth, session, головне меню, message-хендлер нотаток, старт планувальника, bot.start()
keyboards.js  — inline-клавіатури (головне меню, список менеджерів, вибір періоду)
stats.js      — фіча 2 (статистика + нотатки)
archive.js    — фіча 3 (архів + аудіо)
report.js     — фіча 1 (планувальник + генерація/відправка звіту)
analyze.js    — OpenAI-резюме по менеджеру для звіту (промпт-заглушка, клієнт дасть свій)
time.js       — Kyiv-час: слоти звітів + межі періодів (день/тиждень/місяць/квартал)
ui.js         — розбиття довгих повідомлень (>4096) + markdown→plain fallback
```

## Деплой

Окремий Render **Background Worker** з того самого репо/образу, старт-команда `node src/bot/index.js`. Env: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `BOT_ALLOWED_CHAT_IDS`, `BOT_REPORT_CHAT_ID`, `BOT_REPORT_TIMES` + спільні `DATABASE_URL`, `OPENAI_API_KEY`, `BINOTEL_*`. Деталі — `../../DEPLOY.md`.
